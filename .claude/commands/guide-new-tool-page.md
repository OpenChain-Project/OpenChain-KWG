# 새 도구 페이지 신규 작성

## 실행 순서

1. tools/1-fossology/_index.md 를 읽어 기준 구조를 파악한다
2. tools/ 디렉토리의 현재 파일 목록을 확인하여 다음 weight 번호를 결정한다
3. 아래 정보를 순서대로 질문한다:
   - 도구명 (영문)
   - 한 줄 설명 (한국어)
   - 주요 역할: SBOM 생성 / 소스코드 스캔 / 취약점 분석 / BOM 관리 중 선택
   - GitHub 저장소 URL
   - 공식 문서 URL
   - 라이선스
   - 지원 언어 또는 플랫폼
4. 모든 정보를 수집한 후 기준 구조에 맞춰 초안을 작성한다
5. 초안을 보여주고 승인을 요청한다
6. 승인 후 파일을 생성하고 TODO.md의 해당 항목을 [x]로 업데이트한다
7. hugo --minify 로 빌드 오류가 없는지 확인한다
