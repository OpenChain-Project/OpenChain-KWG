# ISO 입증자료 충족 여부 점검

## 사용법
/project:guide-verify-evidence 5230 3.1.1.1
/project:guide-verify-evidence 18974 4.3.2.1

## 실행 순서

1. 인자로 받은 표준(5230 또는 18974)과 입증자료 번호를 확인한다

2. content/ko/guide/.claude/reference/ 에서 해당 입증자료 원문을 읽는다

3. 해당 입증자료를 충족할 수 있는 관련 파일을 아래 기준으로 결정하고 읽는다:
   - iso{표준번호}_guide/ 의 대응 조항 페이지
   - opensource_for_enterprise/ 의 대응 섹션
   - templates/1-policy/ 또는 templates/2-process-template/

4. 아래 기준으로 충족 여부를 판단한다:
   - ✅ 충족: 입증자료가 요구하는 내용이 가이드 또는 템플릿에 명시되어 있고,
             실제 기업이 이 가이드를 따르면 입증자료를 제출할 수 있는 수준
   - ⚠️ 부분 충족: 관련 내용은 있으나 입증자료 요건을 완전히 충족하기에는
                  설명이 부족하거나 샘플이 없는 경우
   - ❌ 누락: 관련 내용이 어디에도 없는 경우

5. 아래 형식으로 결과를 보고한다:

---
**입증자료 {번호}**
> {입증자료 원문 (KO)}

**충족 여부**: ✅ / ⚠️ / ❌

**근거 파일**:
- {파일 경로}: {충족하는 내용 요약 1~2줄}

**부족한 부분** (⚠️ 또는 ❌인 경우):
- {무엇이 없는지 구체적으로}

**권장 조치** (⚠️ 또는 ❌인 경우):
- {어느 파일의 어느 섹션에 무엇을 추가해야 하는지}
---

6. 결과를 content/ko/guide/EVIDENCE-CHECK.md 에 누적 기록한다.
   파일이 없으면 새로 생성하고, 있으면 해당 항목만 추가/업데이트한다.
