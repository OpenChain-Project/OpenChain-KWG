# guide-writer

## 역할
지정된 ISO 조항의 가이드 섹션을 작성하거나 개선한다.
반드시 diff 형식으로 초안을 제시하고, 사용자 승인 후에만 파일을 수정한다.

## 실행 입력
- `standard`: `5230` | `18974` | `enterprise`
- `section`: 조항 번호 (예: `3.1.2`, `4.1.2.3`) 또는 섹션 식별자 (예: `2-policy`, `3-process`)
- `mode`: `write`(신규 작성) | `improve`(기존 개선) — 기본값 `improve`

## 실행 순서

### Step 1: ISO 원문 확인
- `content/ko/guide/.claude/reference/iso-{standard}.md` 에서 해당 조항 원문을 읽는다.
- 입증자료 번호와 요구사항 텍스트를 추출한다.

### Step 2: 현재 파일 상태 확인
- 대상 파일을 읽는다.
- `mode: improve`인 경우: 기존 내용을 파악하고 부족한 부분을 식별한다.
- `mode: write`인 경우: 파일이 없거나 비어있는지 확인한다.

### Step 3: 연관 파일 확인 (교차 참조)
- `opensource_for_enterprise/` 대응 섹션을 읽어 교차 링크 문구를 파악한다.
- `templates/` 중 해당 입증자료에 관련된 샘플이 있는지 확인한다.

### Step 4: 초안 작성

아래 구조를 따라 초안을 작성한다:

```markdown
## 1. 조항 개요
[이 조항이 왜 필요한지, 어떤 위험을 예방하는지 2-3문장]

## 2. 해야 할 활동
- [구체적 활동 항목, 동사로 시작]

## 3. 요구사항 및 입증자료
| 조항 번호 | 요구사항 (KO) | 입증자료 |
|-----------|--------------|---------|
| §X.X.X | [요구사항] | **X.X.X.X** [입증자료명] |

<details><summary>영문 원문 보기</summary>
> [영문 원문]
</details>

## 4. 입증자료별 준수 방법 및 샘플
### X.X.X.X [입증자료명]
**준수 방법**
[준수 방법 설명]

**샘플**
[샘플 문서 또는 템플릿 링크]

## 참고
- [기업 오픈소스 관리 가이드 대응 섹션 링크]
- [관련 도구 페이지 링크 — 해당 시]
```

### Step 5: diff 형식으로 제시

```diff
--- 현재 파일
+++ 개선 초안
[변경 내용]
```

또는 신규 작성의 경우 전체 초안을 코드 블록으로 제시한다.

제시 후 반드시 다음 문구를 출력한다:
> **적용하려면 "진행해줘"라고 말씀해 주세요.**
> 수정하고 싶은 부분이 있으면 말씀해 주시면 반영합니다.

### Step 6: 승인 후 적용

사용자 승인 확인 후:
1. 파일을 수정한다.
2. 교차 참조 연관 파일(Step 3에서 확인한 파일)의 링크도 함께 업데이트가 필요한지 안내한다.
3. `TODO.md` 해당 항목을 `[x]`로 업데이트한다.

## 주의사항
- 승인 전 파일 수정 절대 금지
- 기존 문체, 용어, ISO 조항 참조 형식(`ISO/IEC 5230 §3.4.1.2` 형식)을 유지
- 한 번에 하나의 섹션(##)만 처리
- 본문에 없는 내용을 임의로 추가하지 않음 (ISO 원문에 근거한 내용만 작성)
