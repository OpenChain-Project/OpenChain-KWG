# 가이드 작성 규칙

가이드 관련 작업 시 항상 적용되는 규칙.
대상 디렉토리: `content/ko/guide/` (archive/ 제외)

---

## 파일 읽기 — 토큰 효율화

- 파일 읽기는 목적에 따라 범위를 최소화한다.
  - 구조 파악 → 프론트매터 + 첫 번째 `##` 제목까지만 읽기
  - 조항 내용 확인 → 해당 섹션(##)만 읽기
  - ISO 레퍼런스 → `content/ko/guide/.claude/reference/` 에서 필요한 조항 번호만 발췌
- 본문 전체를 읽기 전에 먼저 구조를 파악하고, 필요한 섹션만 읽는다.

---

## 파일 수정 — 승인 절차

- 파일 수정 전 반드시 diff 형식으로 변경 초안을 먼저 제시한다.
- 사용자 승인("진행해줘", "ok", "적용해") 없이는 어떤 파일도 수정·생성하지 않는다.
- 한 번에 하나의 섹션(##)만 처리한다.
- 기존 문서의 문체, 용어, ISO 조항 참조 형식을 그대로 따른다.

---

## 교차 참조 일관성 — 수정 시 영향 범위 확인

수정 대상 파일에 따라 아래 연관 파일을 반드시 확인한다.

| 수정 대상 | 확인해야 할 연관 파일 |
|-----------|----------------------|
| `iso5230_guide/` 조항 | `opensource_for_enterprise/` 대응 섹션의 교차 링크 |
| `iso18974_guide/` 조항 | `opensource_for_enterprise/` 대응 섹션 + 준용 여부(ISO 5230 대응 조항) |
| `opensource_for_enterprise/` 섹션 | 대응 ISO 조항 가이드의 "참고" 링크 |
| `templates/1-policy/` | 연관 입증자료 조항 가이드의 "샘플" 섹션 |
| `templates/2-process-template/` | 연관 입증자료 조항 가이드의 "샘플" 섹션 |
| `tools/` 페이지 | `opensource_for_enterprise/4-tool` 의 해당 도구 링크 |

## 가이드 간 의존 관계 (요약)

- **ISO 18974 전용 항목** (ISO 5230 준용 없음): `4.1.2.3`, `4.1.2.5`, `4.1.2.6`, `4.1.4.2`, `4.1.4.3`, `4.1.5.1`, `4.2.2.3`, `4.3.2.1`, `4.3.2.2`
- ISO 18974 나머지 항목은 ISO 5230 대응 조항을 준용하므로, 5230 가이드 수정 시 18974 가이드도 영향권에 포함됨
- Templates는 다수 입증자료의 샘플 기반: 템플릿 수정 시 연관 조항 가이드의 샘플 섹션과 정합성 유지

---

## 작성 품질 기준

- 각 조항 페이지는 다음 순서를 따른다:
  1. 조항 개요 (왜 이 조항이 필요한가)
  2. 해야 할 활동
  3. 요구사항 및 입증자료 표
  4. 입증자료별 준수 방법 및 샘플
  5. 기업 오픈소스 관리 가이드 연결 링크
- 입증자료 번호는 원문 그대로 사용 (예: `3.1.1.1`, `4.1.2.3`)
- ISO 조항 번호 병기 형식: `ISO/IEC 5230 §3.4.1.2`
- 내부 링크는 상대 경로 사용
